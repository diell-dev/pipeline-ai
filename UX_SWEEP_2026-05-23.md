# Pipeline AI — UX Sweep (2026-05-23)

Quick walkthrough of the deployed app at https://pipeline-ai-beige.vercel.app on both desktop (1440×900) and mobile (380×800). Findings ranked by user impact, with rough effort. Numbered for cross-referencing.

**Pages reviewed:** Dashboard · Jobs · Schedule · Equipment (already) · Clients · Proposals · Invoices · Finances · Settings.

---

## CRITICAL — feels broken / risks data loss / blocks task

### 1. Trash icon in invoice rows has no confirm
**Where:** `/invoices` — each row has a red trash icon next to "Mark Paid"
**Why it matters:** One missed click deletes an invoice. No "are you sure?" intercept visible. Combined with rows looking near-identical (five $136.09 rows in a row), the wrong-row-click risk is real.
**Fix:** Either move destructive actions to a row context menu (kebab/ellipsis), or wrap delete in a Confirm dialog. Effort: small.

### 2. Bottom nav covers page content on mobile
**Where:** Every page on mobile. The "Pending Review" filter chip on the Jobs page got partially clipped by the fixed bottom nav. Same problem will hit any scroll-to-bottom interaction (last list row, last form field).
**Why it matters:** Field techs lose access to actions below the fold.
**Fix:** Add `pb-20` (or whatever your bottom-nav height + safe inset is) to the main scroll container on mobile. Effort: tiny — one className change.

### 3. Dashboard "Revenue This Month -100% vs last month" reads as catastrophic
**Where:** `/finances` and the dashboard equivalent KPI strip. Today is May 24, revenue this month is $0 (because no paid invoices yet this month), and the card shows `-100% vs last month` in red.
**Why it matters:** Looks like the business collapsed. The framing isn't useful when the month just started or when the previous baseline was tiny.
**Fix:** Either drop the percentage-change line in the first week of the month, or only show comparison when the absolute change is meaningful (e.g., previous month > $X and current month < threshold). Effort: small.

## HIGH — clearly improves the daily experience

### 4. Schedule "Week" view has no time axis
**Where:** `/schedule` with Week selected. It's a 7-column day grid (Mon–Sun) with events as small floating cards, no hours of the day. Looks more like a Month view than a Week view.
**Why it matters:** Dispatchers can't see WHEN within a day a job is scheduled, only that it exists on that day. Defeats the purpose of Week view.
**Fix:** Either build a proper 7×hours grid (most calendar libraries support this), or rename current to "Week summary" and add a real "Week" view. Effort: medium.

### 5. Schedule date label is awkward — "May 18 – 2026 (day: 24)"
**Where:** `/schedule` header.
**Why it matters:** Reads like a parsing error. Should be a normal date range.
**Fix:** Render as "Week of May 18, 2026" or "May 18 – 24, 2026". Effort: tiny.

### 6. Jobs status filter counts don't add up
**Where:** `/jobs` — chips show All 12, Pending Review 0, Revision Requested 1, Scheduled 1, Approved 3, Sent 5. Sum = 10, but All = 12.
**Why it matters:** Two jobs are in statuses (probably Completed, Cancelled, or Submitted) that have no chip. Users with those jobs can't filter to them, which is confusing.
**Fix:** Either add the missing status chips, or hide the empty ones (Pending Review = 0 right now and isn't useful as a clickable filter). Effort: small.

### 7. /invoices and /finances both show essentially the same invoice list
**Where:** Two nearly identical tables of the same 12 invoices. /finances has 4 KPI cards above; /invoices has a status filter and "Filter by client" search. Lots of overlap.
**Why it matters:** Users won't know which to use. Twin pages also mean two places to update when the schema changes.
**Fix:** Define a clear split — e.g., /finances = analytics + KPIs + read-only summary, /invoices = the operational list with row actions. Or merge into one page with collapsible KPI strip. Effort: medium.

### 8. Settings doesn't include Team or Subscription/Billing
**Where:** `/settings` cards: Profile, Company Profile, Email Integration, Payments, Notifications, Security. Team is in the sidebar as a separate top-level nav item. Subscription/Billing doesn't exist anywhere.
**Why it matters:** Org admins expect Team management under Settings (it's their settings → people). Subscription will be table-stakes once self-serve billing exists.
**Fix:** Add Team management card to Settings (or sub-nav inside Settings). Stub a Subscription card with "Coming soon" or wait until billing ships. Effort: small for nav restructure.

### 9. Filter-chip active state inconsistent
**Where:** `/jobs` shows "All 12" as solid black/dark, "Revision Requested 1" as yellow-outlined. Different selection styles.
**Why it matters:** Users can't tell which chip is currently active at a glance.
**Fix:** Pick one selection style and apply to all (most likely: solid dark for active, outlined for inactive). Effort: tiny.

### 10. "Full Operations" tag at the top of every page is mystery meat
**Where:** Top-left of every page header, in a pill-shaped tag. Not interactive. Same text on every page.
**Why it matters:** Users will wonder what it does/means. If it's the tenant/site context, label it as such.
**Fix:** Either prefix with "Tenant:" / "Org:", or hide when only one option exists, or remove. Effort: tiny.

## MEDIUM — polish that compounds

### 11. Dashboard KPI number colors don't map to anything
**Where:** Dashboard hero cards — "12" in black, "0" in blue, "0" in orange, "8" in green.
**Why it matters:** Users assume colors mean something (blue=info, orange=warning, green=good) but here they're arbitrary.
**Fix:** Either pin colors to status semantics (orange for "Pending Review" because it needs attention, green for "Approved", etc.) or make all the same accent color. Effort: tiny.

### 12. Dashboard "Avg Started → Completed" is orphaned on its own row
**Where:** Second row of KPIs has 4 cards (mostly "—"), then a third row with just one card.
**Why it matters:** Visual weight is wrong. The orphan card draws disproportionate attention to "no data".
**Fix:** 5-column grid at this width, or hide the orphan when value is empty. Effort: small.

### 13. Quick Actions buttons on dashboard have inconsistent hierarchy
**Where:** Dashboard bottom — "+ New Proposal" (filled dark), then "+ Submit New Job", "View Clients", "View All Jobs" (all outline).
**Why it matters:** Filled = primary action. Why is "New Proposal" the only primary among siblings of equal weight?
**Fix:** Either all secondary outline, or rank explicitly: 1 primary (probably "+ New Job"), then secondaries. Effort: tiny.

### 14. "Pipeline AI — Super Admin" subtitle duplicates the brand
**Where:** Dashboard welcome message.
**Why it matters:** The brand name is already in the sidebar. Subtitle is just the role.
**Fix:** "Super Admin" alone, or "Diell · Super Admin", or drop the subtitle. Effort: tiny.

### 15. Equipment Lifecycle empty cards on dashboard
**Where:** 3 cards all showing "0" with descriptions like "0 overdue invoices".
**Why it matters:** Empty zeros take up visual space and signal nothing.
**Fix:** Collapse to a single status line when all zero: "✓ All equipment current". Expand to cards when there's actual data. Effort: small.

### 16. "Replacement cost by category (top 5)" shows one item with empty bar
**Where:** Dashboard widget. Single category, $0 cost.
**Why it matters:** Looks broken — empty progress bar with no value.
**Fix:** Hide the widget entirely when there's < 2 items or all values are 0. Effort: tiny.

### 17. Invoice rows visually indistinguishable
**Where:** /invoices and /finances — five rows in a row all showing $136.09 to Manhattan Towers LLC.
**Why it matters:** Hard to tell which invoice is which without reading the invoice #.
**Fix:** Add a one-line subtitle per row (service summary, line item count, e.g. "Drain inspection · 2 items"). Effort: small if data is available, medium if it requires a new join.

### 18. Filter labels redundant ("All Statuses", "All Clients")
**Where:** /finances and /invoices — filters in unselected state show "All Statuses" / "All Clients" as their value.
**Why it matters:** Looks like real selections, not placeholders. Users see "All Statuses" and don't realize they can change it.
**Fix:** Use placeholder styling (lighter text) when value is the default. Effort: tiny.

### 19. Jobs list heavy repetition
**Where:** /jobs — 5 of the 7 visible rows all say "Manhattan Towers LLC" with the same address.
**Why it matters:** Visual noise drowns out the actual variation (date, status, submitter).
**Fix:** Group by client OR show client name only when it changes from the previous row. Effort: small.

### 20. Mobile dashboard filter takes two rows when it could be one
**Where:** Mobile dashboard — TIMEFRAME and CLIENT each on their own line.
**Why it matters:** Eats vertical space that's already scarce on phones.
**Fix:** Side-by-side at this width. Effort: tiny.

## LOW — nice-to-have

### 21. "Mark Paid" appears only for unpaid invoices but "Eye" disappears for paid ones
**Where:** /finances Actions column. Paid row has no eye icon; unpaid rows have Mark Paid + eye.
**Fix:** Keep Eye/View action consistent across all rows; only Mark Paid is conditional.

### 22. "More" in bottom nav is mystery meat on mobile
**Where:** Mobile bottom nav 5th item "More".
**Fix:** Either show first letters of the categorized items (Clients · Invoices · Settings...) or replace "More" with a clearer label.

### 23. Empty clients list has no empty state
**Where:** /clients with 0 clients would just be blank.
**Fix:** Show "No clients yet" empty state with the same pattern as the Proposals empty state.

### 24. Status pill color scheme not documented
**Where:** Across the app — Scheduled is light blue, Sent to Client is green, Approved is darker green, Revision Requested is yellow.
**Fix:** Define a status → color mapping in tokens, apply consistently. Effort: small (we already have design tokens).

### 25. No global search
**Where:** Top bar has only notification bell + user avatar.
**Fix:** Add a global search input (jobs, invoices, clients, equipment) — once the app has enough data to need it. Defer.
